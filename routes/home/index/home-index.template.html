Introduction

In this app we attempt to show...

  • how to achieve a relatively complex set of states in our application while keeping the app well-organized.
  • how to switch states absolutely or relatively in Ionic.
  • how to either share or not share templates irregardless of our tab navigation context (whether or not the current state is loaded inside or outside of Ionic's tabs interface).
  • how to structure states so that navigation can happen properly without needing to know if we're inside a tab or not.
  • how to use sub-sub states like tabs.home.index.
  • and lastly, how to sanely use Angular's flat, string based module system with jspm/systemjs's ES6 module support.

Ionic's special sauce

Ionic takes care of some subtle but important things for us.

Note that if we're navigating into the tab interface from outside of it or if we're navigating internally between tabs then Ionic does not display the back button because it treats each tab section as a separate navigation tree in our app.

If instead, we're outside of tabs, we'll get the back button because links are treated as being inside the same navigation tree.

Built using

In this very simple application we're using:

  • ES6 modules
  • jspm
  • SystemJS
  • Angular 1.x
  • Ionic 1.0.0 beta 14.
